## Правила работы по сессиям (CRITICAL — AI читает первым!)

1. **Одна сессия = один шаг.** Никогда не делать два шага за сессию.
2. В начале сессии AI читает этот файл и определяет следующий невыполненный шаг.
3. AI делает ровно этот шаг: код + тесты + документация (если требуется).
4. После прогона тестов AI обновляет этот файл: ставит ✅ у выполненного шага.
5. AI делает `git add` + `git commit` с сообщением из плана.
6. **В конце сессии AI выдаёт готовый промпт для нового чата** — разработчик копирует его, открывает новый чат, вставляет.
7. Никакой самодеятельности: если шаг неясен — спросить, не гадать.

## Контекст (чтобы не забыть между сессиями)

- **Зачем:** r/hubspot — «HubSpot tells current MRR but not what it was on April 1st 2024. Huge blocker.»
- **Что храним:** агрегаты метрик (НЕ сырой CSV) в Supabase, привязанные к `user_id`.
- **Кто пишет:** только авторизованные пользователи (бесплатные — тоже, это retention-механика).
- **Где код:** новый модуль `app/core/snapshot.py` + изменения в `app/pages/5_dashboard.py` и `app/pages/7_account.py`.
- **Структура таблицы:** `supabase/migrations/v3.3_snapshots.sql` (SQL-миграция).

## Структура таблицы `snapshots` (согласовать на Шаге 1)

```sql
CREATE TABLE IF NOT EXISTS snapshots (
    snapshot_id    UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id        UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    period         TEXT NOT NULL,          -- 'YYYY-MM'
    mrr            FLOAT,
    arr            FLOAT,
    arpu           FLOAT,
    churn_rate     FLOAT,
    nrr            FLOAT,
    ltv            FLOAT,
    active_subscribers INTEGER,
    total_revenue  FLOAT,
    source         TEXT DEFAULT '',        -- имя загруженного файла
    created_at     TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE (user_id, period)
);
```
